O mito do "só modelos 16B+"
O comentário que circula na comunidade é impreciso. A limitação não é o tamanho do modelo — é o contexto. O OpenClaw precisa de mínimo 16k tokens de contexto. O Ollama por defeito usa apenas 4096 tokens, o que faz falhar. A solução é configurar OLLAMA_CONTEXT_LENGTH=16384. Com esta configuração, o Llama 8B funciona.
# Velocidade de inferência comparativa GPU RTX 4090 (24GB): ~100-150 tokens/seg ← fluido e interactivo MacBook M1 Pro: ~30-60 tokens/seg ← excelente para uso local CPU VPS 2 cores: ~5-15 tokens/seg ← aceitável para background CPU VPS 4 cores: ~10-25 tokens/seg ← ok para heartbeats
3 Ollama + OpenClaw + Hostinger
Hostinger suporta Ollama
Template oficial disponível
O Hostinger tem um template Ubuntu 24.04 com Ollama + Llama 3 + Open WebUI pré-instalados. Escolhes no hPanel ao criar o VPS — zero configuração manual.
OpenClaw suporta Ollama
Integração nativa
O OpenClaw integra Ollama nativamente. Detecta automaticamente modelos com suporte a tools via /api/tags. Configuração mínima no openclaw.json.
# Configuração OpenClaw para Ollama (openclaw.json) { "model": { "default": "claude-haiku-4-5", "routing": { "heartbeat": "ollama/llama3.1:8b", ← grátis, local "routine": "ollama/llama3.1:8b", ← grátis, local "complex": "claude-haiku-4-5", ← API barata "advanced": "claude-sonnet-4-5" ← só se necessário } }, "heartbeat": { "provider": "local", "model": "ollama/llama3.1:8b", "interval_minutes": 60 }, "tools": { "byProvider": { "ollama": { "profile": "minimal" } ← reduz overhead de tokens } } }
Configuração crítica — Context Length
O Ollama usa 4096 tokens por defeito. O OpenClaw precisa de ≥16k. Tens de definir explicitamente no VPS:
# No VPS — definir context length do Ollama export OLLAMA_CONTEXT_LENGTH=32768 # Ou via Docker Compose environment: - OLLAMA_HOST=0.0.0.0:11434 - OLLAMA_CONTEXT_LENGTH=32768 # Porquê 32k e não 16k? # System prompt + tool schemas = ~6700 tokens de overhead # 32k deixa ~25k livres para conversa real

GPU VPS 24/7 — Não faz sentido
GPU cloud é cobrado por hora. Para uso contínuo 24/7, o custo mensal é proibitivo ($288–$1.433/mês). Só compensa alugar GPU por horas para tarefas pontuais intensivas.

VPS CPU + API cloud ~$15/mês 5-15 tok/s 3B–8B local Recomendado

GPU VPS por horas $0.30–$0.60/h 100+ tok/s Qualquer Para tarefas pontuais

MacBook M1 Pro — excelente para LLM local
Memória unificada (UMA): CPU e GPU partilham o mesmo pool. Ollama usa Metal nativamente → 30–60 tok/s com Llama 8B Q4. Com 16 GB RAM sobra espaço confortável. Usa o Mac para desenvolvimento e testes locais.

1
Criar VPS Hostinger KVM 2
Seleccionar template "Ubuntu 24.04 com Ollama" no hPanel. Ollama + Llama 3 + Open WebUI já instalados. Configurar OLLAMA_CONTEXT_LENGTH=32768.
2
Instalar OpenClaw no VPS
curl -fsSL https://get.openclaw.ai | bash — Seguir wizard de onboarding. Seleccionar Ollama como modelo local + Claude Haiku como fallback.
3
Fazer pull do Llama 8B Q4
ollama pull llama3.1:8b-instruct-q4_K_M — Modelo principal para heartbeats e tarefas rotineiras. Gratuito, sem custos de API.
4
Configurar model routing
Editar ~/.openclaw/openclaw.json com routing: Ollama para heartbeats/rotina, Haiku para moderado, Sonnet só para complexo.
5
Acesso remoto via Tailscale
Instalar Tailscale no VPS para acesso seguro. Nunca expor a porta 18789 directamente à internet.
Quando escalar para mais?
Se o volume crescer muito → upgrade para KVM 3 (16 GB, ~$12.99) para Llama 8B mais confortável. Se quiseres total privacidade e zero API → Mac Mini M4 Pro (64 GB, ~$1.400 uma vez) é a escolha da comunidade OpenClaw para setup local sério.
